# Дорожная карта

## Правила игры

### Задача

Разработка класса и интерфейса правил игры и структуры
Разработка возможности сохранения правил в разных местах

Нужно иметь возможность создавать правила, сохранять.
Изменять текущие правила, а значит читать их, преобразовывать в нужную структуру.

Сохранять нужно в постоянное хранилище и временное, а также в файл.

### Разбор структуры с постоянным хранилищем

Для подключения к базе данных(это может быть json, postgres, sqlite или redis как кэш) используется самописный контекстный менеджер подключения
ConnectionProtocol - протокол для подключения разного рода баз данных
JsonConnection - менеджер для подключения к json
PgConnection - менеджер для подкючения к postgres

Данные мы будем передавать, валидируя по пути в pydantic
StructGameRules - класс валидации для передачи данных

Интерфейс для передачи должен уметь сохранять данные, выдавать данные по ключу.
Удалять данные, перезаписывать текущие данные.

Написать точку входа в которой будут прописан весть функционал в виде api
