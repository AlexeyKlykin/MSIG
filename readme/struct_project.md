Я попытаюсь равномерно распределить обязанности между разными частями программы.

#game_rules 
Правила игры связывают между собой:
- предметы 
- эффекты 
- игроков
описывают аспекты:
- перемещения по карте 
- получения предметов и эффектов 
- взаимодействия с предметами и другими игроками
и др.

#клетка
Имеет идентификатор. Имеет свойтства такие как:
1. Множественная привязка
2. Временные эффекты
3. Информация о привязке

#карта
Имеет картинку в клетку. Например в клетке есть отметка о высоте и глубине от чего можно будет построить логическое утверждение о возможности местонахождения объектов на карте. Какие то клетки будут верхушкой горы = значит в свойствах будет отметка что на клетке не чего находится не может. и т.д. Описание - [game rules](readme/game_rules) 

#game_engine 
По задумке движок игры является основным механизмом взаимодействия игроков и администраторов с базой данных, правилами игры.
Движок позволяет получать запросы, формировать ответы, производить разного рода расчеты через интерфейс. 

#interface 
Интерфейс позволяет передавать запросы движку через api и отдавать данные полученные от движка для отображения на карте и в сумке игрока. 

#колесо/рандомайзер 
Набор рандомайзеров для получения данных от движка. Данными могут быть предметы, варианты испытаний и т.д.

#монетка 
Бросок монеты - алгоритм определяющий один из двух вариантов. Нужен для определения действия игрока в некоторых случаях. Возможность избежать наложения негативных эффектов.
