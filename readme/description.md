Описание проекта MSIG (Multiplayer Streamer Interactive Game)

Игра делится на ряд основных компонентов:
- Интерфейс пользователя(frontend)
	- Сумка игрока
	- Карта
	- Таблица лидеров
	
- Интерфейс администратора(admin)

- Движок игры(backend)
	- интерактивная система ответов сервера
	- возможность подключения языковой модели
	- алгоритмы выборки произвольных элементов из хранилища по определенным параметрам
	- броски кубиков
	- бросок монетки

- Правила игры(отдельная сущность для хранения ссылок на хранилище)
	- Описания всех сущностей. Возможность хранить в разных форматах
	- Предметы
	- Эффекты
	- Статусы игроков и администраторы
	- и т.д.
	
- API - интерфейс для подключения к движку из вне
	- api для фронта
	- api для общения с базой данных
	
- Хранилище:
	- база данных
	- временное, часто обновляемое хранилище

Цель игры дойти до конца и выиграть в супер игре. Игрок попадая на карту бросает кубики. Перемещает аватар по карте. Выполняет испытания. Использует предметы на себя или другого игрока. Получает эффекты. 

#interface 
К сайту могут подключиться пользователи в одной из четырех ролей.
Роли: незарегистрированный пользователь, администратор, пользователь, игрок.
Пользователь регистрируется на сайте. Он получает роль пользователя и попадает в личный кабинет. В личном кабинете он может подать заявку на смену роли.

#пользователь
Пользователи могут:
1. видеть доступные игры 
2. просматривать карты 
3. видеть инвентарь игроков.

#игрок
 В личном кабинете пользователь может подать заявку на роль игрока и выбор карты. После подтверждения от администрации он может выбрать:
 1. аватар персонажа
 2. специализацию
 3. союзников

#admin
Через панель администратора админ может править правила игры. Подтверждать регистрацию пользователей. Менять роли пользователей

#inventory
Выбрав карту, игрок получает доступ к выбору аватара и своему инвентарю. В инвентаре находится описание, количество очков, ресурсы, эффекты, доступные испытания, предметы и т.д. При наведении курсора на элемент инвентаря выпадает ссылка на правила игры с описанием этого элемента.

#maps_game 
На каждой карте существует лимит игроков зависящий от величины карты. 
Карта состоит из матрицы чисел. Каждое число это клетка.
Клетка является объектом с рядом свойств таких как:
- активная или пассивная(может ли игрок на нее встать)
- имеет привязку к области
- могут находится предметы и ловушки
- может быть переходом
Область это(горы, болота, снег, пустыня и т.д.):
- набор клеток
- набор свойств и эффектов
Одновременно на клетке могут находится несколько игроков. С клетками можно взаимодействовать. По клеткам может передвигаться аватар игрока.
У карты есть окно свойств:
- там отображаются игроки
- место игроков на карте и по удалению друг от друга
- статус игрока
- бафы и дебафы игрока

#game_rules
Правила игры описывают все аспекты игры. На правила есть ссылки у всех элементов игры. В правилах описано буквально все:
- взаимодействия игроков
- приоритет эффектов
- применение предметов
- и т.д.
Правила игры хранятся в базе данных и могут быть изменены только администраторами.

#game_engine
Игрок посылает запрос, через #api игровому движку, на проведение того или иного действия(получение эффекта, выбор испытания и т.д.). Движок делает запрос к базе данных с правилами в которых описаны все возможные взаимодействия, и генерирует ответ игроку по определенному алгоритму.

#effects
Эффекты имеют положительные и отрицательные свойства. Могут быть наложены на персонажа игрока или персонажа соперника. Эффекты могут быть получены в результате перемещения по карте или от применения предметов.

#items
Предметы могут быть получены путем перемещения по карте. Украдены у других игроков. Из сундуков в процессе рандомизации движком игры. Могут накладывать эффекты или иметь прямое взаимодействие(например толчок игрока на шаг назад или вперед).
